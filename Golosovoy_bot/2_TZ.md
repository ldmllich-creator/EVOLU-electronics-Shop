# 📋 Техническое задание: Голосовой AI-бот для интернет-магазина

## 1. Общее описание

Разработка голосового AI-бота, подключаемого к телефонной линии интернет-магазина. Бот принимает входящие звонки, распознаёт речь клиента, обрабатывает запрос с помощью ИИ и отвечает голосом в реальном времени.

## 2. Функциональные требования

### 2.1 Приём и обработка звонков
- Подключение к виртуальному номеру телефона
- Автоматический ответ на входящий звонок
- Приветствие клиента (настраиваемое)
- Многоканальность — обработка нескольких звонков одновременно

### 2.2 Распознавание и синтез речи
- **Speech-to-Text (STT)** — распознавание речи клиента в реальном время
- **Text-to-Speech (TTS)** — озвучивание ответов бота натуральным голосом
- Поддержка русского языка
- Допустимая задержка ответа: не более 2-3 секунд

### 2.3 Обработка запросов ИИ
- Подключение к LLM (Claude / GPT / другая модель через OpenRouter)
- Контекстное окно на весь разговор — бот помнит, что говорилось ранее
- Работа на основе загруженных инструкций (базы знаний):
  - Описания товаров
  - Инструкции по эксплуатации
  - FAQ
  - Политика возврата и гарантии
  - Прайс-лист

### 2.4 Консультация по товарам
- Ответы на вопросы: «Что это?», «Сколько стоит?», «Есть в наличии?»
- Сравнение товаров между собой
- Рекомендации на основе потребностей клиента
- Информация о доставке и оплате

### 2.5 Техподдержка и претензии
- Пошаговая диагностика проблем с товаром
- Классификация обращения: вопрос / жалоба / возврат / гарантийный случай
- Инструкции по устранению типичных проблем
- Фиксация сути претензии

### 2.6 Логирование
- Запись всех звонков (аудио)
- Автоматическая транскрибация разговоров
- Сохранение в базу данных:
  - Дата/время звонка
  - Номер клиента
  - Тема обращения
  - Результат (решён / переведён на оператора / перезвонить)
  - Полная транскрибация
- Панель администратора для просмотра логов

### 2.7 Переключение на оператора
- Автоматическое определение, когда нужен человек:
  - Клиент просит переключить
  - Бот не может решить проблему (3+ неудачных попытки)
  - Агрессивное поведение клиента
  - Запрос выходит за рамки базы знаний
- Передача контекста разговора оператору
- Уведомление оператора (мессенджер / email)

## 3. Нефункциональные требования

- **Доступность:** 24/7/365
- **Задержка ответа:** < 3 секунд
- **Масштабируемость:** от 1 до 50 одновременных звонков
- **Язык:** русский
- **Безопасность:** шифрование данных, защита персональных данных клиентов

## 4. Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Среда разработки | Antigravity |
| Телефония | Twilio / Voximplant / виртуальный номер |
| Speech-to-Text | Whisper API / Google STT |
| Text-to-Speech | ElevenLabs / Google TTS / Yandex SpeechKit |
| LLM (мозг бота) | Claude Opus через OpenRouter |
| База данных | Supabase |
| Backend | Node.js + Express |
| Frontend (админка) | React / HTML + JS |
| Деплой | Vercel / VPS сервер |

## 5. Входные данные (что нужно загрузить боту)

- Каталог товаров (названия, описания, цены, наличие)
- Инструкции по эксплуатации товаров
- FAQ (часто задаваемые вопросы и ответы)
- Политика возврата и гарантии
- Скрипты разговоров (приветствие, прощание, переключение)
- Контакты оператора для переключения

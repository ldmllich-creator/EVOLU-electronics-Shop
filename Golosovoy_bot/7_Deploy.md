# 🌐 Деплой (размещение на сервере)

## Что такое деплой
Деплой — это когда мы переносим проект с нашего компьютера на удалённый сервер, чтобы бот работал 24/7, даже когда наш компьютер выключен.

## Варианты деплоя

### Вариант 1: VPS-сервер (рекомендуемый)
- **Что:** арендуем виртуальный сервер
- **Где:** DigitalOcean, Hetzner, Timeweb Cloud, Selectel
- **Стоимость:** от $5-15/мес
- **Плюсы:** полный контроль, стабильная работа телефонии
- **Минусы:** нужно настраивать

### Вариант 2: Vercel (для админ-панели)
- **Что:** бесплатный хостинг для веб-приложений
- **Плюсы:** бесплатно, автоматический деплой
- **Минусы:** не подходит для телефонии (только для админки)

### Вариант 3: Railway
- **Что:** облачная платформа для backend
- **Стоимость:** от $5/мес
- **Плюсы:** простой деплой, подходит для backend
- **Минусы:** лимиты на бесплатном тарифе

## Порядок деплоя

### Шаг 1. Подготовка
- [ ] Убедиться, что проект работает локально без ошибок
- [ ] Все API-ключи вынесены в переменные окружения (.env)
- [ ] Проект загружен на GitHub

### Шаг 2. Настройка сервера
- [ ] Арендовать VPS (Ubuntu 22.04)
- [ ] Установить Node.js на сервер
- [ ] Клонировать проект с GitHub
- [ ] Установить зависимости (npm install)
- [ ] Настроить переменные окружения на сервере

### Шаг 3. Запуск
- [ ] Настроить PM2 (менеджер процессов) для автозапуска
- [ ] Настроить Nginx (прокси-сервер) для админ-панели
- [ ] Настроить SSL-сертификат (HTTPS)
- [ ] Привязать домен (при наличии)

### Шаг 4. Подключение телефонии
- [ ] Настроить вебхуки Twilio → адрес сервера
- [ ] Проверить приём звонков на сервере
- [ ] Тестовый звонок через реальную телефонную сеть

### Шаг 5. Проверка
- [ ] Бот отвечает на звонки 24/7
- [ ] Админ-панель доступна по HTTPS
- [ ] Логи пишутся корректно
- [ ] Мониторинг работает (уведомления о падении)

# 🏗️ Архитектура проекта: Голосовой AI-бот

## Общая схема работы

```
Клиент звонит → Телефония (Twilio/Voximplant)
    → Speech-to-Text (распознаёт речь)
        → LLM (Claude — формирует ответ)
            → Text-to-Speech (озвучивает ответ)
                → Клиент слышит ответ

Параллельно: всё логируется → База данных (Supabase) → Админ-панель
```

## Компоненты системы

### 1. Модуль телефонии
- Приём входящих звонков через API провайдера
- Управление сессиями (начало/конец звонка)
- Переключение на оператора

### 2. Модуль распознавания речи (STT)
- Потоковое распознавание речи клиента
- Конвертация аудио → текст в реальном времени

### 3. Модуль ИИ (LLM)
- Получает текст от STT
- Ищет релевантную информацию в базе знаний
- Формирует ответ с учётом контекста всего разговора
- Определяет тип обращения и необходимость переключения

### 4. Модуль синтеза речи (TTS)
- Конвертация текстового ответа → аудио
- Натуральное звучание голоса
- Минимальная задержка

### 5. Модуль логирования
- Запись аудио звонков
- Сохранение транскрибаций
- Классификация обращений
- Ведение истории клиентов

### 6. Админ-панель
- Просмотр всех звонков
- Чтение транскрибаций
- Статистика обращений
- Управление базой знаний

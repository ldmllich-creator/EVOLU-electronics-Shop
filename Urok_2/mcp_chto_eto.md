# 🔌 MCP — Model Context Protocol

**Источник:** Конспект Урока 2 (Поток 15, 05.03.2026)

---

## Что такое MCP?

**MCP (Model Context Protocol)** — это протокол, созданный компанией **Anthropic** (создатели Claude). Он позволяет ИИ-агенту **напрямую подключаться к внешним сервисам** (базы данных, дизайн-инструменты, фреймворки) и работать с ними **без написания кода подключения**.

### Простая аналогия:

> Представь, что ИИ-агент — это **секретарь**. Раньше, чтобы он достал данные из базы, тебе нужно было написать ему **подробную инструкцию на каждый сервис** (это API). Теперь MCP — это как **универсальный пропуск**: один раз подключил, и секретарь сам знает, куда зайти и что достать.

---

## Как это работает?

```
Ты ставишь задачу
       ↓
  Агент (Antigravity)
       ↓
  MCP-сервер  →  подключается к сервису (Supabase, Figma, ...)
       ↓
  Получает JSON с данными
       ↓
  Агент возвращает тебе результат
```

Тебе **не нужно** знать, как устроено подключение. MCP сам находит нужные данные.

---

## API vs MCP — в чём разница?

| | API (старый подход) | MCP (новый подход) |
|---|---|---|
| **Подключение** | Пишешь отдельный код для каждого сервиса | Одно подключение — и всё работает |
| **Настройка** | Ключи, миграции, SQL-запросы | Один запрос агенту |
| **Ошибки** | Часто | Практически нет |
| **Где работает** | На продакшне и в разработке | ⚠️ **Только в разработке** (пока!) |

> **Важно:** MCP пока работает только **при разработке** (внутри Antigravity). На готовом продукте (продакшн) пока используются обычные API.

---

## Пример: Supabase без MCP и с MCP

### ❌ Без MCP (много ручной работы):
1. Заходишь в Supabase → авторизуешься
2. Пишешь SQL-код для создания таблиц
3. Запускаешь миграцию
4. Проверяешь таблицы
5. Повторяешь для каждого изменения...

### ✅ С MCP (автоматически):
1. Один раз подключил MCP-сервер Supabase
2. Говоришь агенту: *«Создай таблицу пользователей с полями имя, email, телефон»*
3. Агент **сам** заходит в Supabase и всё делает

## Как установить MCP-сервер?

### Способ 1 — Попросить Antigravity:
> *«Я хочу продвинуто разрабатывать. Помоги мне установить все необходимые MCP-серверы.»*

### Способ 2 — Через Context7:
1. Зайти на [context7.com](https://context7.com)
2. Скинуть ссылку в чат Antigravity
3. Попросить подключиться

### Где хранится настройка:
Файл `.gemini/settings.json` в папке проекта. Пример:
```json
{
    "mcpServers": {
        "context7": {
            "command": "npx",
            "args": ["-y", "@upstash/context7-mcp@latest"]
        },
        "supabase": {
            "command": "npx",
            "args": ["-y", "supabase-mcp-server@latest"],
            "env": {
                "SUPABASE_URL": "твой_url_из_supabase",
                "SUPABASE_SERVICE_ROLE_KEY": "твой_ключ_из_supabase"
            }
        }
    }
}
```

## 📝 Итого — запомни главное

1. **MCP = умный мост** между ИИ-агентом и внешними сервисами
2. Не нужно писать код подключения — **всё автоматически**
3. Пока работает **только при разработке** (не на продакшне)
4. Настройка лежит в `.gemini/settings.json`
5. Если нужен новый сервис — ищи MCP-сервер на [context7.com](https://context7.com) или [skills.sh](https://skills.sh)

# 🛠️ Регламент разработки IT-продуктов (Vibe Coding)

**Версия:** 2.0 (Уроки 1–2)  
**Курс:** Вайбкодинг, Поток 15  
**Последнее обновление:** 05.03.2026

---

## 📌 Основной принцип

Создание цифровых продуктов **без глубокого программирования** — с помощью ИИ-агента (Antigravity). Единственный критерий: **работает или не работает**.

> **Формула успеха:**  
> 40–60% — качество документации (шаги 1–5)  
> 20% — сама разработка  
> 40% — доработка и исправление ошибок

---

## 📌 Как пользоваться этим регламентом

1. **Перетащите** этот файл в чат Antigravity при начале нового проекта
2. Напишите: *«Используй этот регламент для разработки»*
3. **Следуйте шагам** — копируйте шаблоны, заполняйте, просите ИИ доработать
4. **Не перескакивайте** шаги — идите последовательно

> ⚠️ **Главное правило:** Пока не закончили шаги 1–5, **не просите писать код**. Добавляйте фразу: *«Не пиши код, только документацию»*

# ЧАСТЬ 1. ЦИКЛ РАЗРАБОТКИ

---

## 🔄 Обзор цикла (9+1 шагов)

```
 0. Исследование   →  Анализ рынка, конкурентов, потребностей
 1. Идея            →  Что за продукт, для кого, зачем
 2. ТЗ              →  Функциональность, поведение, границы
 3. Архитектура     →  Структура файлов, технологии, связи
 4. Дизайн          →  Внешний вид (UI/UX), макеты
 5. Задачи          →  Чеклист для ИИ
    5.1 Запуск      →  Локальная работа на компьютере
 6. Тестирование    →  Поиск и фиксация багов
 7. Деплой          →  Публикация на сервер
 8. Дебаггинг       →  Исправление ошибок на сервере
 9. Продакшн        →  Финальный релиз
```

### ⚠️ Правила работы с циклом:
- Возвращаться на предыдущие шаги — **нормально**. Не бояться переделывать
- Если идея сырая, продукт всё равно получится — потом дебажим
- Если всё ломается — **удалить папку и начать заново** (это быстро)
- На шаге 1 (Идея): просите ИИ **структурировать** идею, а **не упрощать и не изменять**

## 💻 Antigravity — среда разработки

### Интерфейс:
| Элемент | Где | Назначение |
|---------|-----|------------|
| Чат с ИИ | Справа | Запросы, промпты, разработка |
| Файлы | Слева | Дерево файлов проекта |
| Просмотр | По центру | Предпросмотр кода и результата |
| Терминал | View → Terminal | Команды, установка, запуск |
| Accept All | Кнопка | Принять изменения |
| Reject All | Кнопка | Откатить изменения |

### Правила:
- 📁 Файлы и папки — **только латиница** с нижними подчёркиваниями (особенно на Windows!)
- 📎 Перетаскивайте файлы/папки в чат — ИИ увидит содержимое
- 🛡️ Не снимайте ограничители — подтверждайте действия вручную
- 📝 Пишите «Не пиши код» — пока формируете документацию
- ⚙️ Settings → Customization → Manage — глобальные правила для ИИ

### Терминал:
- **New Terminal** — привязан к текущему проекту (папке)
- **New Window** — новое окно Antigravity (другой проект)
- Команда запуска проекта (**npm run** / **start** / др.) — запомнить и записать

## 🧩 Skills (Навыки / Субагенты)

### Что это:
Субагенты — маленькие роботы-помощники, которые делают основного агента Antigravity **умнее** в конкретных областях.

### Принцип работы:
```
Вы ставите задачу → Основной агент → Выбирает нужный скилл
                                        ↓
                               Скилл выполняет работу
                               (копирайтинг, SEO, 
                                разработка, дизайн...)
```

### Каталог: [skills.sh](https://skills.sh) — 84 000+ скиллов

### Установка:
1. Найти скилл на skills.sh
2. Скопировать команду установки (вверху страницы скилла)
3. Вставить в **терминал** Antigravity
4. Выбрать: **в проект** (локально) или **глобально** (на уровне ОС)

### Типы установки:
| Тип | Видимость | Рекомендация |
|-----|-----------|-------------|
| **В проект** | Папка с файлами в директории | Специфичные скиллы |
| **Глобально** | Скрыто, работает везде | FindSkills и универсальные |

### Рекомендованная стратегия:
1. Установить **FindSkills** глобально (скилл ищет другие скиллы)
2. Открыть проект
3. Написать: «Изучи мой проект и подбери нужные скиллы»
4. FindSkills подберёт рекомендации → установить их в проект

### Управление:
- **Удалить из проекта:** удалить папку из файловой директории
- **Посмотреть глобальные:** спросить у Antigravity или использовать команду в терминале
- **Удалить глобальный:** команда в терминале (спросить у Antigravity)

### Скиллы vs Плагины:
- **Скиллы** — Markdown-файлы с кодовой базой, ИИ ссылается как на правила
- **Плагины** — программные расширения Antigravity с визуализацией (~50 штук)

## 🔌 MCP (Model Context Protocol)

### Что это:
Протокол от Anthropic — **умный аналог API**. Позволяет агенту напрямую работать с внешними сервисами без ручного кодирования.

### Как работает:
```
Вы даёте задачу → Агент → MCP-сервер → Сервис (Supabase/Figma/...)
                                              ↓
                                        JSON с данными
                                              ↓
                                   Агент ← обработанный результат
```

MCP **сам** находит нужные данные в сервисе — вам не нужно писать код подключения.

### API vs MCP:
| Аспект | API | MCP |
|--------|------|------|
| Подключение | Отдельный код для каждого сервиса | Одно подключение ко всему |
| Настройка | Ключи, миграции, SQL | Один запрос агенту |
| Ошибки | Частые | Практически нет |
| Применение | Разработка + продакшн | Пока только разработка |

### Полезные MCP-серверы:
| Сервер | Назначение |
|--------|------------|
| **Supabase** | База данных без ручной миграции |
| **Figma** | Дизайн и макеты |
| **Next.js** | Фронтенд-фреймворк для сайтов |
| **Context7** | Документация и библиотеки |

### Как установить:

**Способ 1 — Через Antigravity:**
> «Я хочу продвинуто разрабатывать. Помоги мне установить все необходимые MCP-серверы. У меня Antigravity.»

**Способ 2 — Через Context7:**
1. Перейти на [context7.com](https://context7.com)
2. Скинуть ссылку в Antigravity
3. Попросить подключиться

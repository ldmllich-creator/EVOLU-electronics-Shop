## 📖 Тренажёр → Регламент

Концепция переименована: **тренажёр** теперь называется **регламент**.

**Как это работает:**
1. Прошли лекцию → транскрибировали → сделали **регламент**
2. Прошли ещё лекцию → сделали ещё один регламент
3. Объединили (мердж) → получили обновлённый, более полный регламент
4. Этот файл **всегда подтягиваете** к новым проектам

**Промпт для создания регламента:**
> «Мы прошли урок по вайбкодингу. В нём описаны важные знания и инструменты, которые мне понадобятся при создании веб-продуктов и других IT-продуктов. Я хочу, чтобы ты сделал полноценный рабочий документ, на который я буду ссылаться при разработке тех или иных продуктов.»

**Где разместить регламент:**
- **Вариант 1:** Файл в проекте — перетаскиваете в чат при работе
- **Вариант 2:** Settings → Customization → Manage — вставить как глобальные правила Antigravity

## 🧩 Skills (Навыки / Субагенты)

### Что это?
Skills — вспомогательные инструменты (субагенты), которые помогают основному агенту Antigravity выполнять задачи лучше и точнее. Уже **84 000+** готовых скиллов.

### Как это работает:
```
Основной агент (Antigravity)
  ├── Скилл: Копирайтер
  ├── Скилл: Фронтенд-разработчик
  ├── Скилл: SEO-продвижение
  └── Скилл: Поиск навыков (FindSkills)
```
Агент сам выбирает, какому субагенту делегировать задачу.

### Где искать: [skills.sh](https://skills.sh)
- 84 000+ скиллов
- Фильтры: по тренду, популярности, категориям
- Каждый скилл — не промпт, а полноценный инструмент с кодовой базой

### Как установить:
1. Находите нужный скилл на skills.sh
2. Копируете команду установки (вверху страницы скилла)
3. Вставляете в **терминал** Antigravity (View → Terminal)
4. Выбираете: **в проект** или **глобально**

| Установка | Где живёт | Когда использовать |
|-----------|-----------|-------------------|
| **В проект** | Папка проекта (видно слева) | Специфичные для проекта |
| **Глобально** | На уровне ОС / Antigravity | Универсальные (FindSkills) |

### ⭐ FindSkills — скилл для поиска скиллов
- Установите **глобально** — он найдёт нужные скиллы для вашего проекта
- Попросите: «Изучи мой проект и предложи нужные скиллы»

### Как удалить скилл:
- **Из проекта** — удалить папку из файловой директории
- **Глобально** — команда в терминале (можно спросить у Antigravity)

### Лайфхак от Радамира:
> Установил FindSkills → попросил изучить проект и предложить скиллы → получил 9 рекомендаций → скинул файл с рекомендациями и написал «Установи все эти скиллы локально в проект» → всё установилось в 2 шага.

## 🔌 MCP (Model Context Protocol)

### Что это?
MCP — **протокол контекста моделей**, созданный Anthropic (создатели Claude). Аналог API, но умнее и проще.

### API vs MCP:

| | API (старый подход) | MCP (новый подход) |
|---|---|---|
| **Подключение** | Отдельный код для каждого сервиса | Одно подключение ко всему сразу |
| **Настройка** | Много строчек кода, ключи, миграции | Один запрос агенту |
| **Ошибки** | Часто | Практически нет |
| **Где работает** | На продакшне и в разработке | **Только в разработке** (пока) |

### Как работает:
```
Агент → запрос → MCP-сервер → сервис (Supabase/Figma/...) 
                                    ↓
                              JSON с данными
                                    ↓
                              Агент ← результат
```

MCP сам понимает, к какому сервису подключиться и какие данные вытащить.

### Где искать MCP-серверы:
1. **В Antigravity** — справа есть раздел MCP-серверов
2. **[context7.com](https://context7.com)** — каталог лучших MCP-серверов
3. **skills.sh** — тоже есть MCP-серверы

### Как установить:
**Способ 1:** Написать в Antigravity:
> «Я хочу продвинуто разрабатывать. Помоги мне установить все необходимые MCP-серверы»

**Способ 2:** Скинуть ссылку на context7.com и попросить подключиться

### Полезные MCP-серверы:
| Сервер | Зачем |
|--------|-------|
| **Supabase** | База данных — без ручной миграции |
| **Figma** | Дизайн через MCP |
| **Next.js** | Фронтенд-фреймворк |
| **Context7** | Универсальный доступ к документации |

### Пример пользы (Supabase):
- **Без MCP:** Заходишь в Supabase → авторизуешься → копируешь SQL-код → запускаешь миграцию → проверяешь таблицы → повторяешь…
- **С MCP:** Один раз подключил → Antigravity сам хозяйничает в Supabase

## 📝 Дополнительные заметки

### Терминал vs Новое окно:
- **New Terminal** — привязан к текущему проекту (папке)
- **New Window** — новое окно Antigravity (можно открыть другой проект)

### Скиллы vs Плагины:
- **Скиллы** — Markdown-файлы, на которые ссылается Antigravity (как промпты с кодовой базой)
- **Плагины** — программные расширения Antigravity с визуализацией

### Мердж:
> **Мердж** (merge) — объединение данных. Запоминайте слово, будете использовать.

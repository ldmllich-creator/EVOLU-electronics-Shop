## 🆚 Cloud Code vs Antigravity

| | **Antigravity** | **Cloud Code** |
|---|---|---|
| **Интерфейс** | Графический (GUI) | Терминал (CLI) |
| **Модель** | Opus 4.6 | Opus 4.6 (та же) |
| **Скиллы** | Есть, но не всегда подхватывает | Автоматически подключает нужные |
| **Мультиагентность** | Ограниченная | Полноценная (5, 10, 15+ агентов) |
| **Откат изменений** | Accept/Reject All | ⚠️ Нет отката — изменения необратимы |
| **Удобство для новичка** | ✅ Проще | Требует опыт в промптах |
| **Подписка** | Своя ($20/мес) | Отдельная ($20 — Pro, $100 — Max) |
| **Расход токенов** | Включён в подписку | Включён в подписку (при работе с ПК) |
| **Проблемы с терминалом** | Бывают (кнопка Run) | Нет |

### 📌 Ключевой вывод:
- **Antigravity** = рабочая лошадка для разработки
- **Cloud Code** = персональный помощник / аудитор / Джарвис
- Можно **использовать оба** одновременно: Antigravity разрабатывает, Cloud Code проверяет

## 🤖 Мультиагентная система (рой агентов)

Самая мощная фишка Cloud Code — запуск **нескольких агентов параллельно**:

```
Пример запроса:
"Запусти 5 агентов:
1. Найди потенциальные баги
2. Оценки процент покрытия кода
3. Опиши функционал проекта
4. Проверь, что нужно доработать
5. Сделай оценку как full-stack разработчик"
```

- Агенты работают **параллельно**, каждый в своей песочнице
- У каждого своя сессия и свой чат
- Результаты приходят к вам в общий чат
- Чем мощнее компьютер — тем больше агентов можно запустить

### 💡 Идея «Анатолий» (субагент-начальник):
> Можно создать главного агента (напр. «Анатолий»), у которого в подчинении команда: охотник за багами, аналитик покрытия, документатор, ревьюер, оценщик. Говоришь «Анатолий, проверь проект» — и вся команда работает.

## 🛠️ Скиллы v2.1 (два уровня)

### Уровень 1 — Улучшающие скиллы
- Текстовые инструкции (MD-файл) с пошаговыми действиями
- Улучшают то, что Cloud Code и так умеет
- **Умная система**: если Opus справляется сам — скилл не подключится (и даже удалится)
- При выходе Opus 4.7/5.0 многие скиллы просто отвалятся (не понадобятся)

### Уровень 2 — Самообучающиеся скиллы
- После выполнения задачи вы даёте обратную связь
- Скилл **улучшается** с каждой итерацией
- На 10–20 раз задача выполняется идеально
- Идеально для **повторяющихся задач**: КП, презентации, отчёты

### Скиллы vs Агенты (субагенты):
| | **Скиллы** | **Агенты/Субагенты** |
|---|---|---|
| Содержимое | Текстовый промпт (MD) | Контейнер: инструкции + API + MCP |
| Сложность | Простые инструкции | Готовая инфраструктура |
| Пример | «Делай ревью по таким правилам» | Полный модуль аналитики с подключениями |

### Глобальные vs Локальные скиллы:
- **Глобальные** (`~/.claude/skills/`) — работают во всех проектах (универсальные: аудит кода, безопасность)
- **Локальные** (в папке проекта) — только для конкретного проекта (спец. скиллы для дизайна, маркетинга)
- **Правило**: если специфика работы сильно меняется между проектами → делайте скиллы локальными

## 🔧 Skill Creator

Встроенный инструмент Cloud Code для создания скиллов:

1. Вызывается через: `/skill` → Skill Creator
2. Задаёте задачу: «Создай скилл для проверки кода»
3. Он автоматически создаёт **5 вариантов** скилла
4. Тестирует каждый + сравнивает с чистым Cloud Code
5. Выбирает лучший (ставит баллы, как A/B-тест)
6. Согласовывает с вами результат

## ⚠️ Важные предупреждения

1. **Cloud Code не откатывает изменения** — в отличие от Antigravity (Accept/Reject), изменения необратимы. Пишите: *«Не пиши код, только объясни»*
2. **Токены тратятся** при автоматизации через телефон/Telegram. С компьютера — в рамках подписки
3. **Скиллы проверяйте** перед установкой — смотрите звёзды, отзывы, профиль
4. **Задачи формулируйте чётко** — иначе сожжёте токены впустую
5. **Температура модели**: высокая = креатив, низкая = точность. Но лучше хороший промпт, чем настройка температуры

## 🔑 Полезные команды Cloud Code

| Команда | Что делает |
|---------|-----------|
| `claude` | Запуск Cloud Code в терминале |
| `/model` | Переключить модель (Sonnet → Opus) |
| `/skill` | Открыть Skill Creator |
| `/plugin` | Установить плагин |
| `Ctrl+C` | Остановить текущую задачу |
| Косая черта `/` | Меню команд и настроек |

## 💬 Вопросы участников и ответы

**Q: Можно ли управлять проектами с телефона?**  
A: Да, через Telegram-бот + Cloud Code (= «Cloud Clo»). Но каждый запрос тратит токены.

**Q: Как переключать модели в OpenRouter?**  
A: Просто скажите в чате: «Хочу работать на Opus 4.6» — агент сам переключит.

**Q: Температура — нужно ли настраивать?**  
A: Лучше написать хороший промпт. Температура: высокая = креатив, низкая = работа с документами.

**Q: Нужен ли отдельный ключ для Whisper (OpenAI)?**  
A: Да, Whisper нет в OpenRouter. Можно использовать Groq (бесплатно, нужен VPN).

**Q: Vercel не принимает обновления?**  
A: Поменяйте модель на Opus. Для приложений с БД лучше деплоить на Timeweb, а не Vercel.

# 🏋️ Тренажёр разработки продуктов

**Используй этот файл каждый раз, когда начинаешь новый проект.**  
Просто следуй шагам — копируй шаблоны, заполняй, и проси Antigravity помогать на каждом этапе.

---

## 📌 Как пользоваться этим тренажёром

1. **Скопируй** нужный шаблон в новый файл для своего проекта
2. **Заполни** каждый раздел — можно своими словами, можно голосом через Whisper
3. **Перетащи** заполненный файл в чат Antigravity
4. **Попроси ИИ** доработать, расширить и детализировать
5. **Двигайся** по шагам — не перескакивай!

> ⚠️ **Главное правило:** Пока не закончил шаги 1–5, **не пиши в промпте ничего про код**. Добавляй фразу: *«Не пиши код, только документацию»*

---

## 🔬 Шаг 0 — Исследование

Прежде чем начать, пойми рынок. Попроси ИИ помочь:

### Промпт для Antigravity:
```
Я хочу создать [тип продукта] для [целевая аудитория].
Проведи анализ:
1. Какие аналоги уже существуют?
2. Какие проблемы пользователей они не решают?
3. В чём может быть моё конкурентное преимущество?
4. Какой минимальный функционал нужен для MVP?

Не пиши код, только аналитику.
```

### Чеклист:
- [ ] Определены конкуренты
- [ ] Найдены незакрытые потребности
- [ ] Сформулировано УТП (уникальное торговое предложение)

## 💡 Шаг 1 — Идея

### Шаблон:
```markdown
# Идея проекта

## Название: [Название продукта]

## Проблема:
[Какую проблему решает? Кому больно без этого продукта?]

## Решение:
[Что именно делает продукт? В 2-3 предложениях]

## Целевая аудитория:
[Кто будет пользоваться? Возраст, профессия, страна]

## Монетизация:
[Как будет зарабатывать? Подписка / разовая оплата / реклама / freemium]

## Примеры использования:
1. [Пользователь заходит и...]
2. [Потом он...]
3. [В результате получает...]
```

### Промпт для Antigravity:
```
Вот моя идея проекта: [вставь заполненный шаблон]

Доработай её:
1. Уточни формулировки
2. Предложи дополнительные фичи
3. Оцени сложность реализации (1-10)

Не пиши код, только документацию.
```

### Чеклист:
- [ ] Проблема описана чётко
- [ ] Решение конкретное (не абстрактное)
- [ ] Понятно, кто пользователь
- [ ] Есть хотя бы 1 путь монетизации

## 📐 Шаг 2 — Техническое задание (ТЗ)

### Шаблон:
```markdown
# Техническое задание

## 1. Общее описание
[Краткое описание проекта и его целей]

## 2. Функциональные требования
### Обязательные (MVP):
- [ ] [Функция 1 — что делает]
- [ ] [Функция 2 — что делает]
- [ ] [Функция 3 — что делает]

### Желательные (v2):
- [ ] [Функция 4]
- [ ] [Функция 5]

## 3. Нефункциональные требования
- Язык интерфейса: [русский / английский / оба]
- Платформы: [веб / мобайл / десктоп]
- Время загрузки: [< 3 сек]
- Максимум пользователей: [100 / 1000 / 10000]

## 4. Интеграции
- [ ] [С какими сервисами нужна интеграция? API, базы, CRM]

## 5. Ограничения
- [Бюджет / сроки / технические ограничения]
```

### Промпт для Antigravity:
```
Вот моя идея: [перетащи файл идеи]

На основе этой идеи создай подробное техническое задание.
Включи: функциональные требования, нефункциональные требования,
список интеграций, и ограничения.

Не пиши код, только ТЗ.
```

### Чеклист:
- [ ] Все функции перечислены
- [ ] Разделены MVP и «потом»
- [ ] Указаны платформы и язык
- [ ] Указаны интеграции (если есть)

├── 📁 assets/

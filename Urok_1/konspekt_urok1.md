## 🎯 Главная идея урока

> **Vibe Coding** — создание цифровых продуктов без глубоких знаний программирования, с помощью ИИ-агента (Antigravity). Единственный критерий: **работает или не работает**.

---

## 🔄 Цикл разработки (9 + 1 шагов)

Это **основа всей работы**. Всегда один и тот же порядок, неважно — бот, сайт, приложение или калькулятор:

| Шаг | Название | Что делаем |
|-----|----------|------------|
| 0 | **Исследование** | Анализ рынка, потребностей (с помощью ИИ) |
| 1 | **Идея** | Описываем: что за продукт, для кого, зачем |
| 2 | **ТЗ** (техническое задание) | Все функции, требования, ограничения |
| 3 | **Архитектура** | Структура файлов и папок проекта |
| 4 | **Требования к дизайну** | Внешний вид: кнопки, цвета, страницы |
| 5 | **Задачи** | Чеклист для ИИ: что делать по пунктам |
| 5.1 | **Запуск локально** | Разработка + запуск на компьютере |
| 6 | **Тестирование** | Ищем ошибки, проверяем работу |
| 7 | **Деплой** | Публикация на сервер |
| 8 | **Дебаггинг** | Исправление ошибок на сервере |
| 9 | **Продакшн / Релиз** | Финальный запуск |

### ⚠️ Формула успеха:
- **40–60%** успеха — качественная документация (шаги 1–5)
- **20%** — сама разработка
- **40%** — доработка и исправление ошибок

### 📌 Важные правила:
- Пока не закончены шаги 1–5, **не просите писать код** → пишите: *«Не пиши код, только документацию»*
- Возвращаться на предыдущие шаги — **нормально**
- Если всё сломалось — удалить папку и начать заново (это быстро)

## 💻 Интерфейс Antigravity

| Зона | Где | Что делает |
|------|-----|-----------|
| **Чат с ИИ** | Справа | Пишем запросы, промпты |
| **Файлы** | Слева | Дерево файлов проекта |
| **Просмотр** | По центру | Предпросмотр кода и результата |
| **Терминал** | View → Terminal | Команды, установка, запуск |
| **Accept All** | Кнопка | Принять изменения ИИ |
| **Reject All** | Кнопка | Откатить изменения на предыдущий шаг |

### Важные элементы:
- **🔄 Кнопка с кругляшком** — история переписок
- **➕ Плюсик** — создать новый чат
- **Show More** — показать все переписки
- Файлы можно **перетаскивать в чат** — ИИ увидит содержимое
- Зелёный текст = добавляется, красный = удаляется (при изменениях)
- **+14 / -5** под файлом = количество добавленных/удалённых строк

## 🔑 Ключевые принципы

1. **Файлы и папки — только латиница** (особенно на Windows!). Кириллица вызывает ошибки
2. **Один критерий**: работает продукт или не работает. Качество кода — не наша задача
3. **Идею прописывайте максимально детально сами**. Просите ИИ *структурировать*, а не *упрощать*
4. **Не снимайте ограничители** — подтверждайте действия ИИ вручную (кнопками Accept/Reject)
5. **Перетаскивайте файлы в чат** — так ИИ видит контекст
6. **Транскрибируйте записи** через бота **Whisper** в Telegram

## 📂 Тренажёр (= Регламент)

Тренажёр — это рабочий документ, который создаётся из транскрипции урока:

1. Транскрибируете запись → создаёте файл `Лекция.md`
2. Просите ИИ выделить: **домашки**, **конспект**, **тренажёр** (= регламент)
3. С каждым уроком обновляете регламент (мердж)

**Промпт:**
> «Я была на лекции по вайбкодингу. Вот транскрипция. Сделай: 1) все домашние задания, 2) конспект, 3) тренажёр для разработки продуктов»

## 🛡️ Безопасность (10 пунктов от Георгия)

1. ИИ **не автономен** — вы решаете, что ему можно
2. Он **не удалит** файлы с компьютера без подтверждения
3. Работает в **отдельных папках** проекта
4. **Проверяйте** промежуточные результаты
5. Глобальные правила: Settings → Customization → Manage
6. Можно написать ИИ: *«Ничего не удаляй без моего подтверждения»*
7. Можно попросить: *«Всё расписывай, что ты делаешь»*
8. Можно развернуть на **отдельном компьютере/виртуальной машине**
9. Делайте **бэкапы** (GitHub, клонирование папок)
10. Специальные **навыки безопасности** можно установить

## 💡 Пример из урока: Калькулятор шкафа

Георгий показал полный цикл на примере **калькулятора стоимости шкафа**:

1. Собрали идею из устного описания замерщика (квадратура, фасады, ящики, штанги, монтаж)
2. Транскрибировали голосовое описание через Whisper
3. Вставили в чат Antigravity → ИИ создал файлы: Идея → ТЗ → Архитектура → Дизайн → Задачи
4. Запустили разработку → ИИ сам написал код и запустил локально
5. Подключили ИИ (Opus) → калькулятор стал «умным» (сам считает, генерирует PDF)
6. **Результат**: рабочий веб-калькулятор с чатом ИИ за ~30 минут
